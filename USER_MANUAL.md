# 👮 คู่มือการใช้งาน Smart Police Report Bot (ฉบับสมบูรณ์)

ระบบนี้ถูกออกแบบมาเพื่อบันทึกข้อมูลรายงานตำรวจลงใน Google Sheets โดยอัตโนมัติผ่าน Telegram โดยเน้นความเสถียร (Stability) และความยืดหยุ่น (Extensibility) เป็นหลัก

---

## 🚀 1. การจัดการระบบ (Management)
ใช้สคริปต์ `./manage.sh` ในการควบคุมการทำงานทั้งหมดผ่านระบบ **tmux** (ทำให้บอทรันเบื้องหลังได้ตลอดเวลา):

| คำสั่ง | หน้าที่ |
| :--- | :--- |
| `./manage.sh start` | เริ่มทำงานบอทในพื้นหลัง (Background) ผ่าน tmux |
| `./manage.sh logs` | เข้าไปดูหน้าจอการทำงาน (Attach) เพื่อเช็ค Logs หรือข้อผิดพลาด |
| `./manage.sh status` | ตรวจสอบว่าบอทยังรันอยู่หรือไม่ (Active/Inactive) |
| `./manage.sh stop` | หยุดการทำงานของบอทและปิด session ทั้งหมด |

> [!TIP]
> เมื่ออยู่ในหน้าจอ **logs** (tmux): กด `Ctrl + B` แล้วตามด้วย `D` เพื่อออกจากหน้าจอโดยที่บอทยังรันอยู่

---

## 🏛️ 2. โครงสร้างชีทและพิกัดคอลัมน์ (Hardcoded Registry)
บอททราบหน้าที่และลำดับคอลัมน์ของแต่ละชีททันทีจากโค้ด (Registry) เพื่อป้องกันความผิดพลาด:

### 📝 Orders (ชีทข้อสั่งการ)
- **หน้าที่**: บันทึกคำสั่งประชุม, ภารกิจด่วน, มติที่ประชุม
- **คอลัมน์**: `Detail`, `Commander`, `Deadline`, `Status`, `Urgency`, `Note`, `Timestamp`

### 👥 Contacts (ชีทรายชื่อ/กำลังพล)
- **หน้าที่**: ฐานข้อมูลเบอร์โทรศัพท์, ตำแหน่ง, นามเรียกขาน และข้อมูลกำลังพล
- **คอลัมน์**: `Name`, `Position`, `Callsign`, `Phone`, `Agency`, `ID Card`, `Birthday`, `Bank`, `Bank Account`, `Start Date`, `Blood Type`, `Note`, `Timestamp`

### 🚗 Vehicles (ชีทยานพาหนะ)
- **หน้าที่**: ตรวจสอบทะเบียนรถหลวง, เลขโล่, และสถานะการซ่อมบำรุง
- **คอลัมน์**: `ID`, `Shield`, `Plate`, `Brand`, `User`, `Repair`, `Status`, `Note`, `Timestamp`

### 📑 Arrests (ชีทจับกุม)
- **หน้าที่**: สรุปสถิติผลการจับกุมคดีอาญา (รองรับการตรวจจับผู้ต้องหาหลายคนอัตโนมัติ)
- **คอลัมน์**: `วันที่`, `หน่วย (สภ.)`, `ผู้ต้องหา`, `อายุ`, `ข้อหา`, `ของกลาง`, `สถานที่เกิดเหตุ`, `พฤติการณ์`, `ผู้จับกุม`, `ระบุสถานะ`, `Note`, `Timestamp`

## 🛡️ 3. การกู้คืนระบบ (Disaster Recovery)
ระบบถูกเซ็ตให้ทนทานต่อไฟดับและอินเทอร์เน็ตหลุด:

### 🔄 ระบบ Reboot อัตโนมัติ (Auto-start on Boot)
หากต้องการให้บอทเริ่มทำงานเองทันทีที่เปิดเครื่องใหม่ (หลังไฟดับ) ให้รันคำสั่งนี้ใน Linux/WSL Shell เท่านั้น:
```bash
# หากอยู่ใน PowerShell ให้พิมพ์ wsl ก่อน
(crontab -l 2>/dev/null; echo "@reboot cd \"$(pwd)\" && ./manage.sh start") | crontab -
```

### 📶 ระบบกู้คืนอินเทอร์เน็ต (Network Recovery)
- บอทมีระบบ **Auto-Retry** หากเน็ตหลุดบอทจะไม่ดับ แต่จะพยายาม Re-connect เองเมื่อเน็ตกลับมามา
- เมื่อระบบออนไลน์คืนมา บอทจะส่งข้อความแจ้งเตือนหาคุณทาง Telegram ทันที

## 🦾 4. ฟีเจอร์ Stronger Recorder 2.0 (ระบบบันทึกอัจฉริยะ)
บอทเวอร์ชันล่าสุดถูกอัปเกรดให้ "แข็งแกร่ง" ในการถอดรหัสข้อความภาษาไทยที่ซับซ้อน:

### ⚖️ 4.1 ระบบบันทึกการจับกุม (Advanced Arrests)
บอทจะแยกแยะข้อมูลจากรายงานการจับกุมโดยอัตโนมัติ แม้พิมพ์มาแบบข้อความยาวๆ:
- **ผู้ต้องหาหลายคน**: แยกชื่อ (นาย/นาง/น.ส.) และอายุ (อายุ ...) ออกมาเป็นรายคนอัตโนมัติ
- **กรองขยะ**: ฉลาดพอที่จะไม่เอาคำว่า "คดียาเสพติด" หรือ "ผู้ต้องหา:" มาบันทึกเป็นชื่อคน
- **ข้อมูลครบถ้วน**: ดึงข้อมูล **ของกลาง, เลขบัตร ปชช., ชุดจับกุม, ที่อยู่ และสถานที่เกิดเหตุ** ลงคอลัมน์ที่ถูกต้องทันที

### ☕ 4.2 ระบบอัปเดตสถานะงบประมาณ (Expenditure Status)
รองรับการอัปเดตสถานะการเบิกจ่ายรายเดือน เพียงพิมพ์สั้นๆ:
- **คีย์เวิร์ด**: น้ำมัน, ไฟฟ้า, ประปา, อินเทอร์เน็ต (เน็ต), โทรศัพท์, เบี้ยเลี้ยง, OT (โอที)
- **ตัวอย่าง**: "จด ค่าน้ำมัน ม.ค. 69 เบิกจ่ายแล้ว" -> บอทจะไปอัปเดตที่แถว "ค่าน้ำมันเชื้อเพลิง" ในชีททันที

### 📝 4.3 ระบบจดบันทึกด่วน (Natural Memos)
- พิมพ์ **"จด [ข้อความ]"** หรือ **"บันทึก [ข้อความ]"**
- บอทจะตัดคำสั่ง "จด" ออก และเก็บเฉพาะเนื้อหาที่สำคัญลงในชีท Orders
- รองรับการจดภารกิจล่วงหน้า เช่น "จด พรุ่งนี้ 10.00 น. ประชุมที่จังหวัด" -> บอทจะแยกวันและเวลาให้อัตโนมัติ

## 🔔 5. การแจ้งเตือนและยืนยัน (Notifications)
- **Activation**: แจ้งเตือน 🟢 ทุกครั้งที่เปิดเครื่อง/เริ่มบอท
- **Success Confirmation**: ยืนยันทุกครั้งที่บันทึกลงชีท พร้อมสรุปข้อมูลที่ถอดรหัสได้ (เช่น พบผู้ต้องหากี่คน, ข้อหาอะไร)
- **Error Diagnostics**: หากบอทไม่เข้าใจรูปแบบ จะแนะนำวิธีการพิมพ์ที่ถูกต้องทันที
