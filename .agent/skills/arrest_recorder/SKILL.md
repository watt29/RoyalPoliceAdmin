# 🚓 Arrest Recorder Skill (V1.0.0)

## 🎯 วัตถุประสงค์
แปลงข้อมูลรายงานผลการจับกุม (Arrest Report) ที่เขียนในรูปแบบข้อความยาวๆ เป็นข้อมูลโครงสร้าง (Structured Data) เพื่อบันทึกลง Google Sheets และรองรับกรณี "มีผู้ต้องหาหลายคนในการจับกุมครั้งเดียว" โดยทำหน้าที่กระจายข้อมูลข้อหาและของกลางให้กับผู้ต้องหาทุกคนอย่างถูกต้อง

## 🛠️ ความสามารถหลัก (Core Intelligence)

### 1. Multiple Suspects Extraction (ระบบแยกแยะผู้ต้องหาแบบกลุ่ม)
- **ปัญหา**: เมื่อตำรวจส่งรายงานจับกุม มักจะเขียนที่อยู่และข้อหารวมกันไว้ที่เดียว แต่มีรายชื่อผู้ต้องหา 5-10 คน
- **ความฉลาด**: AI จะดึงโครงสร้าง (ผู้ต้องหาคนที่ 1, 2, ... N) ออกมาเป็น Array ย่อย และนำ "วันที่", "ข้อหา", "ของกลาง" ซึ่งเป็นข้อมูลส่วนกลางของการจับกุมครั้งนั้น มาประกบรวมเป็น Record ย่อยให้กับผู้ต้องหาแต่ละคน

### 2. Auto Record Routing (แยกลงชีทคดีเฉพาะ)
- **ปัญหา**: ป้องกันข้อมูลจับกุมปะปนกับรายชื่อบุคคลทั่วไปหรือตำรวจ (Contacts)
- **ความฉลาด**: ตรวจจับคำว่า `จับกุม`, `ผู้ต้องหา`, `ข้อหา` ใน Text และดักจับให้เข้ากระบวนการแยกผลคดี (Arrest Pipeline) เพื่อส่งไปเก็บที่ชีท `Arrests` ทันที

## 📋 รูปแบบข้อมูลที่รองรับ (Schema)
ชีทเป้าหมาย: `Arrests`

ข้อมูล 1 แถวต่อ **ผู้ต้องหา 1 คน** ประกอบด้วย:
- `วันที่จับกุม`: ดึงจากบริบทในข้อความ
- `หน่วยงาน`: เช่น สภ.มหาราช
- `ผู้ต้องหา`: ชื่อ-สกุล ผู้ต้องหา
- `อายุ`: อายุ
- `เลขบัตร ปชช.`: กรณีมีการระบุ
- `ที่อยู่ผู้ต้องหา`: ข้อมูลบ้านเลขที่ต่างๆ
- `ข้อหา`: ฐานความผิด
- `ของกลาง`: สิ่งของที่ยึดได้
- `สถานที่จับกุม`: ที่เกิดเหตุ
- `ชุดจับกุม`: เจ้าหน้าที่ที่เกี่ยวข้อง
- `Timestamp`: เวลาที่บอทบันทึก

## 🔄 กระบวนการทำงาน
1. User ส่งข้อความรายงานแบบยาวเข้า Telegram
2. `ContactHandler` ตรวจพบ Keywords ชุดข้อหาจับกุม
3. ยิงข้อมูลไปหา Google Gemini AI ด้วย Prompt เฉพาะทาง `extract_arrest_record`
4. แปลง JSON Array ที่ได้ แตกแถวออกและทำ Bulk Insert ผ่าน `SheetsService.append_arrests_bulk`
