# 📊 Unit Salary Report Skill (V1.0.0)

## 🎯 วัตถุประสงค์
เพื่อสรุปข้อมูลเงินเดือน ขั้นเงินเดือน และประวัติการพิจารณาผลการประเมิน (ย้อนหลัง 5 ปี) ของเจ้าหน้าที่ทุกคนในสังกัดเดียวกัน (เช่น สภ.มหาราช) โดยแสดงผลในรูปแบบที่อ่านง่ายและเป็นระเบียบ

## 🛠️ ความสามารถหลัก (Core Intelligence)

### 1. Unit Filtering (ระบบกรองรายหน่วย)
- **ตรรกะ**: ค้นหาเจ้าหน้าที่ทุกคนจาก Sheet `Contacts` และ `Commanders` ที่มี "สังกัด" (Agency) ตรงกับหน่วยงานที่ระบุ
- **การเชื่อมโยง**: ดึงข้อมูลเงินเดือนและประวัติประเมินจาก `SalaryHistory` โดยใช้ชื่อ-นามสกุลเป็น Key หลัก

### 2. Auto-Check & Validation (ระบบตรวจสอบแท่งเงินเดือน)
- **ตรวจสอบย้อนกลับ**: เปรียบเทียบ "เงินเดือนปัจจุบัน" กับ "ระดับ/ขั้น" ในบัญชีอ้างอิง `SalaryReference` (บัญชี พ.ร.บ. 2565)
- **สัญลักษณ์สถานะ**:
  - ✅ **ตรงตามแท่ง**: เมื่อยอดเงินเดือนตรงกับบัญชีมาตรฐาน
  - ⚠️ **ไม่ตรงแท่ง**: เมื่อพบความคลาดเคลื่อน พร้อมระบุยอดเงินเดือนที่ควรจะเป็นตามกฎหมาย

### 3. Comprehensive Evaluation History (ประวัติการพิจารณารายปี)
- **ข้อมูลย้อนหลัง**: ดึงข้อมูลจากคอลัมน์ `ประเมินสรุป` ใน `SalaryHistory`
- **การแสดงผล**: แยกรายการเป็นข้อๆ (Bullet Points) ตามปี พ.ศ. เพื่อใช้ประกอบการพิจารณาเลื่อนขั้นในรอบถัดไป

## 📋 รูปแบบการแสดงผล (Display Format)
1. **Header**: ระบุชื่อหน่วยงาน และจำนวนเจ้าหน้าที่ทั้งหมด
2. **Body**: แสดงผลเป็น "บัตรประจำตัว (Person Card)" รายบุคคล เรียงตามลำดับ
3. **Summary**: (ออปชั่น) ตารางสรุปภาพรวมสำหรับหน่วยงานที่มีเจ้าหน้าที่จำนวนมาก

## 🔄 คำสั่งเริ่มใช้งาน
- พิมพ์ **"รายงานเงินเดือน [ชื่อหน่วย]"** หรือ **"สรุปพิจารณา [ชื่อหน่วย]"**
- ระบบจะดึงข้อมูลทุกคนในสังกัดนั้นมาแสดงผลทันที
