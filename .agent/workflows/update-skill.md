# 🔄 Workflow: การอัปเดตและจัดการเวอร์ชัน Skill

เพื่อให้การพัฒนาบอทมีความเสถียรและสามารถย้อนกลับ (Rollback) ได้เมื่อเกิดข้อผิดพลาด ให้ปฏิบัติตามขั้นตอนดังนี้:

## 1. การเตรียมการ (Pre-Update)
- ตรวจสอบเวอร์ชันปัจจุบันใน `SKILL.md` (YAML Frontmatter)
- วิเคราะห์ความต้องการใหม่และผลกระทบต่อตรรกะเดิมใน `scripts/`

## 2. การอัปเดตไฟล์ (Implementation)
- **Update Logic**: แก้ไขโค้ดในโฟลเดอร์ `scripts/` ของสกิลนั้นๆ
- **Update Reference**: หากมีรูปแบบข้อมูลใหม่ ให้เพิ่มนิยามใน `references/data_schemas.md`
- **Update Version**: ปรับเลขเวอร์ชันใน `SKILL.md` (เช่น จาก 1.0.0 เป็น 1.1.0)

## 3. บันทึกการเปลี่ยนแปลง (Changelog)
- เพิ่มบันทึกในไฟล์ `CHANGELOG.md` ภายในโฟลเดอร์สกิล เพื่อระบุว่า:
  - เพิ่มความสามารถอะไร?
  - แก้ไข Bug ตรงไหน?
  - ใครเป็นผู้ปรับปรุง?

## 4. การทดสอบและนำไปใช้ (Deployment)
- รันคำสั่งทดสอบ (ถ้ามี) หรือลองส่งข้อมูลตัวอย่างผ่าน Telegram
- **สำคัญ**: หากมีการเปลี่ยนตรรกะในไฟล์ Python ให้ทำการ Restart Bot ทุกครั้ง
// turbo
```powershell
Get-Process python* | Stop-Process -Force; Start-Sleep 2; python main_bot.py
```

## 5. การ Rollback (กรณีเกิดข้อผิดพลาด)
- หากเวอร์ชันใหม่ทำงานผิดพลาด ให้ย้อนกลับไปใช้ Code จาก Snapshot ก่อนหน้า หรือใช้ Git Revert ไฟล์ในโฟลเดอร์สกิลนั้นๆ
